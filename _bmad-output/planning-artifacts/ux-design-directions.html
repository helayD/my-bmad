my-bmad 设计方向集

Step 9 · 6 个控制面方向

基于品牌蓝-青-绿渐变、Linear + GitHub 灵感、以及“委托系统持续推进工作”的核心体验，下面是 6 个完整方向。

01 控制塔 Direction信号优先 / 任务全局态 02 工程工作台 Direction列表-详情-证据链 03 运行时间线 Direction执行流 / 恢复流中心 04 Review Hub Direction产出审查 / 决策沉淀 05 Command Grid Direction高密度仪表 / 团队治理 06 Focus Queue Direction个人推进 / 待响应优先

默认推荐

  • 推荐主方向：01 控制塔 Direction
  • 最佳组合：01 的总体布局 + 02 的详情页结构 + 03 的时间线反馈
  • 适合 my-bmad 的原因：可信、清晰、可治理、可持续扩展
Design Direction Explorer

为 `my-bmad` 生成的 6 组控制面视觉方向

每个方向都围绕同一个目标：让用户把研发目标交给系统后，既能感到系统真的在推进，又能在任何时刻快速理解当前状态、风险、交互请求与接管路径。差异主要体现在信息层级、导航组织、视觉密度和“控制感”的表达方式上。

最清晰 01 控制塔 Direction，适合打造产品默认主方向。
最工程化 02 工程工作台 Direction，更像 Linear + GitHub 的结合体。
最适合个人模式 06 Focus Queue Direction，最强调“现在我该处理什么”。

适合 my-bmad 的原因

它最符合“前台极简、后台严控”的产品目标。用户先看到的是系统当前是否可靠、哪里需要处理，而不是复杂模型本身。

  • 适合个人与团队共用同一控制面语言。
  • 最有利于建立“状态可信度”。
  • 方便后续扩展风险、治理、审批、审计模块。

特点

更适合熟悉 issue / PR / checks 模型的团队。用户会感觉这是一个更工程化、更证据导向的工作台。

  • 适合团队协作与审查流。
  • 略弱于控制塔方向的“全局掌控感”。

建议的锁定策略： 以 01 控制塔 Direction 作为默认主方向，吸收 02 的工程详情结构与 03 的执行时间线表达。这样最贴合 my-bmad 的差异化价值：既像产品控制面一样清晰，又像工程系统一样可信。
