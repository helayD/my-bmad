# Story 1.6: 团队级执行与治理策略配置

Status: done

## Story

作为组织管理员，
我希望能配置团队工作空间的默认执行策略、治理策略和控制规则，
以便团队内所有项目和任务在统一治理框架下执行。

## 验收标准

1. **Given** 管理员进入团队工作空间设置页面 **When** 管理员查看执行策略配置区域 **Then** 显示可配置项：默认 agent 路由偏好、并发任务上限、自动恢复策略开关、审批节点策略 **And** 每项配置附带说明和当前值

2. **Given** 管理员修改某项治理策略 **When** 提交配置变更 **Then** 系统校验配置值合法性（Zod 4 校验） **And** 配置保存到 `Workspace.settings` JSON 字段 **And** 变更记录审计事件（操作人、变更前值、变更后值）

3. **Given** 管理员配置了"所有任务执行前需审批"策略 **When** 团队成员发起任务 **Then** 任务进入"待审批"状态而不是直接派发（**注意**：任务执行系统在 Epic 4/5，本 Story 仅保存配置，不实现派发拦截逻辑）

4. **Given** 配置表单包含高风险项（如关闭自动恢复） **When** 管理员提交该项变更 **Then** 系统在提交前展示影响范围摘要（UX-DR19）

5. **Given** 仅团队工作空间（`type === "TEAM"`）存在策略配置 **When** 管理员访问个人工作空间设置 **Then** 执行策略配置区域不显示（或页面不存在）

6. **Given** 非管理员用户（MEMBER/VIEWER/AUDITOR）访问设置页 **When** 尝试提交配置变更 **Then** Server Action 拒绝并返回权限不足错误

## Dev Notes

### 关键架构约束

- **`Workspace.settings` 是 `Json?` 字段**（Prisma schema 已存在，Story 1.1 建立）。本 Story 不需要 schema 迁移，仅在该字段中读写 `WorkspaceGovernanceSettings` JSON 对象。
- **Prisma Json 写入**：写入时需要强转 `settings as unknown as Prisma.InputJsonValue`（项目使用 `@/generated/prisma/client`，需从该路径导入 `Prisma` 命名空间）。
- **服务端权限判定**：权限校验**必须**在 Server Action 层执行（与 1.5 一致），中间件只做 session 检查。仅 OWNER/ADMIN 可写，所有角色可读（AC #6）。
- **审计事件 MVP 实现**：Epic 7 的 `AuditEvent` 表尚不存在。本 Story 用 `console.info("[AUDIT]", ...)` 结构化日志满足 AC #2，真实持久化在 Epic 7 补齐。
- **任务派发拦截（AC #3）**：`requireApprovalBeforeExecution: true` 仅需保存配置，**不**在本 Story 实现拦截逻辑（Epic 4/5 范围）。

### settings 字段读写模式

```typescript
// 读取：处理 null 和部分字段缺失

  ...(rawSettings ?? {}),

// 写入
await prisma.workspace.update({
  where: { id: workspaceId },
  data: { settings: newSettings as unknown as Prisma.InputJsonValue },
});
```

### Action 调用流程

```
getWorkspaceSettingsAction(workspaceId) →
  1. getAuthenticatedSession() — 认证
  2. cuid2 校验 workspaceId
  3. getWorkspaceMembership(workspaceId, userId) — 验证成员资格
  4. getGovernanceSettings(workspaceId) → 合并 DEFAULT_GOVERNANCE_SETTINGS + DB 值
  5. return { success: true, data: settings }

updateWorkspaceSettingsAction(input) →
  1. getAuthenticatedSession() — 认证
  2. updateWorkspaceSettingsInputSchema.safeParse(input) — Zod 校验
  3. getWorkspaceMembership(workspaceId, userId) — 获取执行者 membership
  4. 检查 role 为 OWNER 或 ADMIN → 否则 FORBIDDEN
  5. updateWorkspaceSettings({ workspaceId, settings, actorUserId })
     ↓ 领域函数内部：
     5a. 校验 workspace.type === "TEAM"
     5b. 读取 previousSettings（用于审计日志）
     5c. prisma.workspace.update(...)
     5d. console.info("[AUDIT]", { event, workspaceId, actorUserId, previousSettings, newSettings })
  6. revalidatePath(`/workspace/${slug}/settings`)
  7. return { success: true, data: null }
```

### 高风险策略变更确认（AC #4 / UX-DR19）

以下变更在 UI 中需 `<AlertDialog>` 二次确认：
- `autoRecoveryEnabled`: `true → false`（影响：任务异常将不再自动恢复，需手工介入）
- `requireApprovalBeforeExecution`: `false → true`（影响：所有任务派发前需审批，可能影响团队工作效率）

确认对话框文案示例：
- 关闭自动恢复："关闭自动恢复后，执行异常的任务不会自动重试，需管理员手工介入处理。确认关闭？"
- 开启审批节点："开启后，团队成员发起的所有任务都需经过审批才能执行，请确认已准备好审批流程。确认开启？"
